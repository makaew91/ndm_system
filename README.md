# queue broker

Минимальный HTTP-брокер именованных очередей с поддержкой long-poll.

Сервис держит в памяти отдельную FIFO-очередь под каждое имя, не использует
никаких внешних зависимостей кроме стандартной библиотеки Go.

## Запуск

```sh
# из исходников
go run . 8080

# собранный бинарь
go build -o broker .
./broker 8080

# docker
docker compose up --build
```

Порт берётся из единственного аргумента командной строки.

## API

### `PUT /<queue>?v=<message>`

Кладёт `message` в конец очереди `queue`.

| Ответ | Условие |
|-------|---------|
| `200` | сообщение принято |
| `400` | параметр `v` не задан или пустой |

### `GET /<queue>[?timeout=<seconds>]`

Забирает первое сообщение из очереди.

* без `timeout` — мгновенно: либо отдаёт сообщение, либо `404`.
* с `timeout=N` — если очередь пуста, ждёт до `N` секунд появления сообщения; по
  истечении возвращает `404`.

| Ответ | Тело |
|-------|------|
| `200` | сообщение |
| `404` | очередь пуста и таймаут истёк |

Ожидающие получатели обслуживаются в порядке прихода: первый пришедший
читатель получит первое отправленное сообщение.

## Пример

```sh
curl -XPUT 'http://127.0.0.1:8080/pet?v=cat'
curl -XPUT 'http://127.0.0.1:8080/pet?v=dog'
curl 'http://127.0.0.1:8080/pet'              # cat
curl 'http://127.0.0.1:8080/pet'              # dog
curl 'http://127.0.0.1:8080/pet?timeout=2'    # ждёт 2с -> 404
```
